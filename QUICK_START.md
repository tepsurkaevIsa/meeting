# Быстрый старт для работы через интернет

## Вариант 1: ngrok (самый быстрый для тестирования)

1. **Установите ngrok:**
   - Скачайте с https://ngrok.com/download
   - Или через Homebrew: `brew install ngrok`

2. **Запустите сервер:**
   ```bash
   npm start
   ```

3. **В другом терминале запустите ngrok:**
   ```bash
   ngrok http 3000
   ```

4. **Скопируйте HTTPS URL** (например: `https://abc123.ngrok-free.app`)

5. **Откройте URL в браузере** - готово! Теперь можно делиться ссылкой с другими.

**⚠️ Важно:** Бесплатный ngrok требует регистрацию и дает временный URL.

---

## Вариант 2: Railway (бесплатно, постоянный URL)

1. **Зарегистрируйтесь:** https://railway.app (можно через GitHub)

2. **Создайте новый проект:**
   - Нажмите "New Project"
   - Выберите "Deploy from GitHub repo" или "Empty Project"

3. **Если через GitHub:**
   - Подключите репозиторий
   - Railway автоматически определит Node.js
   - Деплой начнется автоматически

4. **Если через CLI:**
   ```bash
   npm install -g @railway/cli
   railway login
   railway init
   railway up
   ```

5. **Получите URL:** Railway автоматически создаст HTTPS URL вида `https://your-app.railway.app`

6. **Готово!** Приложение доступно через интернет.

---

## Вариант 3: Render (бесплатно)

1. **Зарегистрируйтесь:** https://render.com

2. **Создайте Web Service:**
   - New → Web Service
   - Подключите GitHub репозиторий
   - Или загрузите код

3. **Настройки:**
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Environment: `Node`

4. **Деплой:** Render автоматически задеплоит и даст HTTPS URL

---

## После деплоя

1. Откройте ваш HTTPS URL в браузере
2. Разрешите доступ к микрофону
3. Создайте комнату и поделитесь ID с другим пользователем
4. Готово! Звонок работает через интернет.

---

## Проверка работы

- ✅ URL начинается с `https://` (обязательно!)
- ✅ Браузер показывает замок в адресной строке
- ✅ Нет ошибок в консоли (F12)
- ✅ Микрофон запрашивает разрешение

---

## Если не работает

1. **Проверьте HTTPS:** WebRTC требует HTTPS (кроме localhost)
2. **Проверьте консоль браузера:** F12 → Console
3. **Проверьте логи сервера:** В панели управления платформы
4. **См. TROUBLESHOOTING.md** для подробных решений
